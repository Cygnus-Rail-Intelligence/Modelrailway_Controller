**Version 2.1 (16.02.2026)** - Bugfix Release
**Problem:** Relais wurden nur 200ms gepulst und schalteten dann wieder ab.

**Lösung:**
- Relais halten jetzt dauerhaft nach dem Tastendruck
- Toggle-Funktion: Erneuter Tastendruck schaltet Relais wieder AUS
- `pulseRelay()` Funktion entfernt
- Direkte digitalWrite() Befehle implementiert
- Status wird im `abstellgleis[]` Array gespeichert

**Code-Änderungen:**
```cpp
// VORHER (Puls-Modus):
pulseRelay(RELAY_ABSTELLGLEIS_1);

// NACHHER (Dauerhaft mit Toggle):
abstellgleis[0] = !abstellgleis[0];
digitalWrite(RELAY_ABSTELLGLEIS_1, abstellgleis[0] ? LOW : HIGH);
```

**Verhalten:**
- Taste 1 drücken → Relais 1 AN (LOW)  → LED/Gleis eingeschaltet
- Taste 1 erneut → Relais 1 AUS (HIGH) → LED/Gleis ausgeschaltet

### 🖥️ Problem 2: Display zeigt Streifen und flackert
**Problem:** Display flackert, zeigt Streifen, funktioniert nur kurz nach Reset.

**Lösungen implementiert:**

1. **SPI-Geschwindigkeit reduziert**
   ```cpp
   SPI.setClockDivider(SPI_CLOCK_DIV8);  // Vorher: Standard-Speed
   ```

2. **Längere Delays nach Init**
   ```cpp
   delay(500);  // Vor Init
   tft.initR(INITR_BLACKTAB);
   delay(200);  // Nach Init
   tft.setRotation(1);
   delay(100);  // Nach Rotation
   ```

3. **Update-Intervall erhöht**
   ```cpp
   const unsigned long DISPLAY_UPDATE_INTERVAL = 500;  // Vorher: 200ms
   ```

4. **Sanfterer Screen-Clear**
   ```cpp
   tft.fillScreen(ST77XX_BLACK);
   delay(10);  // Kurze Pause nach Clear
   ```

5. **Alternative Init-Optionen dokumentiert**
   - INITR_BLACKTAB (Standard)
   - INITR_GREENTAB (Alternative 1)
   - INITR_REDTAB (Alternative 2)
   - Siehe DISPLAY_TROUBLESHOOTING.txt

## Testanleitung

### Relais-Test (Abstellgleis-Modus)
1. Arduino hochladen und starten
2. Serieller Monitor öffnen (9600 Baud)
3. Licht-Taste drücken → Display zeigt "ABSTELLGLEIS"
4. Taste 1 drücken → Relais 1 klickt AN, LED leuchtet
5. Taste 1 erneut → Relais 1 klickt AUS, LED aus
6. Relais sollte jetzt im AUS-Zustand bleiben
7. Im Display: "Gleis 1: AUS"
8. Wiederholen für Taste 2-4

### Display-Test
1. Arduino hochladen und starten
2. Display sollte zeigen:
   - "CYGNUS-RAIL Controller"
   - "Ready..."
   - Nach 2 Sek: Hauptbildschirm
3. Kein Flackern (max. alle 500ms Update)
4. Keine Streifen oder Artefakte
5. Text klar lesbar

**Falls Display immer noch flackert:**
→ Siehe DISPLAY_TROUBLESHOOTING.txt

## Bekannte Einschränkungen

- Display-Update alle 500ms (reduziert für Stabilität)
- Bei sehr schnellen Tastendrücken kann Display Status verzögert anzeigen
- Relais sollten für Impulsbetrieb geeignet sein (nicht Dauer-24V!)
